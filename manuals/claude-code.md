# Claude Code

## Skills

- [llm-council](https://www.youtube.com/watch?v=92rma4chrIE) : to challenge big questions
- [graphify](https://github.com/safishamsi/graphify) : graph database for llm interactions, reduce token usage and save money
- [gStack](https://github.com/garrytan/gstack) : lot of great skills
- [token-savings](https://github.com/andrew-tenkara/CLAUDE-MD/tree/main/skills/token-savings) : setup rtk and headroom to save tokens and money on llm calls, [article here](https://andrewpatterson.dev/posts/token-savings-rtk-headroom/)

## Tools

- [RTK](https://github.com/rtk-ai/rtk) : proxy that reduces LLM token consumption by 60-90% on common dev commands (seems that token-savings skill is already setup-ing rtk)
- [Headroom](https://github.com/chopratejas/headroom) : compress tool outputs, logs, files, and RAG chunks before they reach the LLM. 60-95% fewer tokens, same answers. Library, proxy, MCP server.
